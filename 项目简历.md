# 工作日志微信小程序

## 项目简介

基于 Taro + React + 微信云开发的多端工作日志记录小程序，支持微信授权登录和账号密码登录两种方式，提供日志增删改查、日历视图、数据统计等核心功能，实现用户级数据隔离，保障数据安全。项目采用 Taro 跨端框架，一套代码可编译到微信小程序、H5 等多个平台。

**项目地址：** https://github.com/muyu32/muyudegithub

## 核心功能

### 1. 用户认证系统
- **微信授权登录**：调用微信登录接口获取 openid，一键登录
- **账号密码登录/注册**：支持用户名密码注册与登录，用户名唯一性校验
- **登录状态管理**：基于 React Context 实现全局登录态，未登录自动跳转登录页
- **首次登录引导**：新用户首次登录展示空日志列表

### 2. 日志管理
- **新增日志**：支持标题、内容、时长、标签等字段，底部弹窗式交互
- **日志列表**：按时间倒序展示，支持关键词搜索（带防抖优化）
- **日志详情**：查看完整日志内容，支持编辑和删除
- **日历视图**：按日期维度查看日志分布，快速跳转指定日期
- **日志删除**：双重校验用户所有权，防止越权操作

### 3. 数据统计
- 总日志数 / 总时长统计
- 今日日志数 / 今日时长统计
- 标签维度统计 Top 5（按使用频次排序）

### 4. 用户数据隔离
- 每条日志绑定 `userId` 字段
- 云函数层严格校验用户身份
- 查询、更新、删除均携带 `userId` 过滤条件
- 确保不同账号数据完全隔离

## 技术亮点

### 1. Taro 跨端开发
- 使用 Taro 4.x 框架，基于 React + TypeScript 编写业务代码
- 一套代码支持编译到微信小程序、H5 等多端
- 利用 Taro 组件库实现跨端 UI 一致性

### 2. 云函数统一封装
- 封装 `callFunction` 统一调用入口，处理小程序/非小程序环境降级
- 统一的 `{ success, data, message }` 返回格式
- 前端业务代码与云函数解耦，便于本地 mock 开发

### 3. 用户数据安全隔离
- 云函数层对所有读写操作强制校验 `userId`
- 删除操作先查询确认归属，再执行删除
- 从架构层面保障多用户场景下的数据安全

### 4. 微信小程序兼容性处理
- 解决 ScrollView 组件在 webview 渲染模式下的 padding 不支持问题
- 修复 Taro 4.x 与微信基础库 3.17.0 兼容性导致的 `_num` 错误
- 替换 calc() 表达式为固定值，避免 CSS 解析异常
- 使用 CSS `overflow-y: auto` 替代 ScrollView 组件，规避框架 Bug
- 自定义底部导航栏替代原生 TabBar，提升交互一致性

### 5. 性能优化
- 搜索功能添加 300ms 防抖，减少频繁查询
- 列表数据按时间倒序排列，支持日期维度快速筛选
- 图片资源使用 SVG 图标，减小包体积

### 6. 工程化规范
- TypeScript 严格类型检查，减少运行时错误
- ESLint + Stylelint 统一代码风格
- SCSS 变量 + CSS Modules 样式隔离
- 模块化组件设计，提高代码复用率
